# 配信参加型RPG 最新実装ロードマップ

## 1. 目的

本ドキュメントは [ideal_rpg_spec.md](ideal_rpg_spec.md) を現行コードへ反映していくための最新版ロードマップを定義する。
2026年3月19日時点の実装状況を前提に、「すでに入っている土台」と「次に着手すべき未完の核」を分けて整理する。

## 2. 現状サマリー

### 2.1 すでに実装済みの土台

- 既存ユーザーデータの後方互換補完
- 装備 `item_id` の安定付与
- 探索履歴の保存と直近 5 件への丸め
- `!前回結果`
- `!履歴`
- `!戦利品`
- `!整理`
- `!保護` と `!保護解除`
- 進行依存の `!攻略`
- エリア初回報酬のデータ定義と受取処理

### 2.2 いま残っている主な論点

- 恒久報酬の数値効果定義を、今後の拡張に向けてさらにデータ駆動へ寄せたい
- OBS 表示は使える状態だが、新要素追加後の整理余地が残っている
- 新エリア、実績、終盤最適化などの中長期拡張は未着手

### 2.3 確認メモ

- 2026年3月19日時点で `.\venv\Scripts\python.exe -m unittest discover -s tests -v` は成功
- 同日、`python dev.py doctor` と `python dev.py test` を入口にすれば `venv` 前提の確認手順を 1 箇所へ寄せられる
- `TWITCH_CHANNEL` は必須化し、`.env.example` から埋める方式に統一した

## 3. 更新方針

- 旧 Phase 0 から Phase 3 は「未着手タスク一覧」としては古くなっているため、今後は完了済み基盤として扱う
- 今後のロードマップは「詰まっている場所」から逆算して並べる
- 中でも自動周回の見え方改善を、次の大型更新の中心課題とする
- 恒久報酬はまず数値効果を小さくてもよいので接続し、その後データ駆動へ広げる
- 新エリアや新素材の追加は、基盤更新後に行う

## 4. フェーズ一覧

- Phase A: 実装済み基盤の維持と棚卸し
- Phase B: 開発環境と運用基盤の固定

- Phase F: 中長期拡張
